В QGIS выделить объекты можно тремя разными способами:

3. Создание буферных зон

3.1. Одним из базовых инструментов геообработки является «Буфер» (в версии 3.2 - Buffer), создающий вокруг исходных объектов полигоны с отступом на заданную дистанцию.

Согласно гипотезе, зона пешей доступности вокруг остановок в зависимости от вида транспорта будет варьироваться между 300, 400 и 500 м. Для визуального отображения этих зон необходимо создать буферные зоны остановок ОТ.

Буферы можно создать для пространственных объектов любого типа геометрии. В данном случае (для точечного объекта) буферной зоной будет окружность. Перед построением буферов критически важно настроить систему координат и проекцию. Для Москвы выберите UTM Zone 37N.

3.2. Перед началом работы с инструментами геообработки крайне важно пересохранить данные в проективную СК, в частности в UTM. В настоящий момент слой с точками остановок разных видов ОТ, для которых необходимо построить буферные зоны, сохранен с использованием географической СК WGS 84, в чем можно убедиться в свойствах слоя на вкладке Information.

В контекстном меню слоя выберите опцию Export — Save Feature As (рис. 2 ⟹ 1), создайте в папке с лабораторной новую папку с произвольным названием (например, “my_shp”) и сохраните туда новый слой, например, railway-stops_utm.shp (рис. 2 ⟹ 2). Не забудьте в качестве СК указать UTM нужной зоны, которая в настоящий момент выставлена для всего проекта (рис. 2 ⟹ 3). Проделайте для всех трех точечных слоев - наземного общественного транспорта, рельсового и метрополитена.

Рис. 2. Перепроецирование shape-файла

3.3. Запустите инструмент Buffer, расположенный в группе инструментов “Вектор”: Vector — Geoprocessing Tools (рис. 3 ⟹ 1). В новом окне Вам предлагается настроить параметры работы инструмента.

Из обязательных параметров Вам необходимо настроить:

  1. класс входных объектов (рис. 3 ⟹ 2), вокруг которых будут строиться буферные зоны — в рамках данного задания точки остановок, пересохраненные в UTM;
  2. дистанция отступа для буферных полигонов — например,  300, 400 или 500 м (какие единицы измерения стоят в настройках инструмента?);
  3. выходной класс геоданных (рис. 3 ⟹ 4) — местоположение нового полигонального слоя с буферными зонами — сохраните данный слой в формате shape-файла в Вашу служебную папку my_shp. Не создавайте временные слои!

Обратите внимание, что многие инструменты геообработки не изменяют исходные данные. На их основе они создают новый набор геоданных, который сохраняется в отдельный shape-файл.

Кроме того, QGIS позволяют строить буферные зоны не только на фиксированной дистанции (как в данном случае), но и на основе значений из поля атрибутивной таблицы — для этого в поле Distance необходимо воспользоваться дополнительным меню — и в нем — выбрав в выпадающем списке нужное поле в разделе Field Type (в старых версиях в том же наборе инструментов существует отдельный инструмент Variable DIstance Buffer). Его использование позволяет варьировать размер буферов в зависимости от значений конкретного показателя в таблице атрибутов.

Попробуйте при выборе класса входных объектов выбрать точки остановок в WGS — что изменится?

3. Создание буферных зон

3.1. Одним из основных инструментов геообработки является «Буфер» (Buffer), создающий вокруг исходных объектов полигоны с отступом на заданную дистанцию.

5.1. Переключитесь в режим компоновки, настройте параметры бумаги (А4, портретная ориентация), экстент.

6.2. Во время процесса компоновки будущей карты удобно пользоваться «направляющими» (Guides) (рис. 6 ⟹ 1), представляющими собой пунктирные линии для позиционирования элементов карты на листе с точностью до 1 мм (по умолчанию). Для их создания щелкните левой кнопкой мыши в любом месте линейки (рис. 6 ⟹ 2).

Обратите внимание, что направляющие доступны только в режиме компоновки. При необходимости автоматическое примагничивание элементов оформления к направляющим можно отключить в свойствах ArcMap (Customize — ArcMap Options — Layout View — Snap element to — Guides). Там же можно настроить прочие вспомогательные элементы режима компоновки.

С помощью направляющих выставите сантиметровые поля со всех сторон листа. В дальнейшем во время оформления карты используйте их для точного позиционирования окна легенды, названия, масштабной линейки и т.д.

6.3. Настройте параметры отрисовки существующих слоев:

  • создайте наглядные значки для остановок различного вида общественного транспорта и настройте их размер;
  • настройте заливку для зоны доступности (например, создайте цветную штриховку — Line Fill Symbols) и её различных категорий;;
  • настройте отрисовку разных типов зданий в соответствии со значением атрибута в поле Type (Layer Properties — Symbology — Categories — Unique Values);
  • настройте отрисовку толщины линий дорог в соответствии со значением атрибута в поле HIGHWAY (Layer Properties — Symbology — Categories — Unique Values) — чем выше иерархический уровень дороги (например, магистраль), тем толще должна быть соответствующая ей линия;
  •  …

6.4. Создайте и оформите легенду, название, масштабную линейку. Для их позиционирования используйте направляющие (см. п. 5.2).

6.5. Добавьте (и оформите!) на лист значение доли жилых зданий, не обеспеченных общественным транспортом, с соответствующей пояснительной надписью (с помощью инструмента «Текст» на панели инструментов «Рисование»).

По завершении данного практического задания:

  • сохраните файл проекта;
  • скопируйте архив со всем проектом на внешний носитель или отправьте себе по почте;
  • экспортируйте получившуюся карту в формат JPEG/PNG и отправьте по почте.

Оптимальный формат названия файла: «06_191_Иванов.jpg», где «06_191» — номер задания и номер подгруппы, фамилия — только кириллицей.

Научитесь пользоваться архиватором

Запомните типы файлов

Запомните типы файлов, используемых в ГИС и их назначение.

Для наших целей мы будем преимущественно использовать векторные данные в формате ESRI Shapefile. Это один из наиболее универсальных форматов, хотя он достаточно старый и имеет ряд серьезных ограничений. Это все еще один из самых распространенных форматов пространственных данных, по аналогии с файлами *.doc для текстовых документов.

ESRI Shapefile

Это не один файл, а набор файлов, которые по отдельности друг от друга бессмысленны и бесполезны. Эти файлы (все с одинаковым именем но разными расширениями) всегда должны храниться и передаваться вместе, потеря одного из файлов равноценна потере всех файлов.

В составе шейпфайла обычно бывают:

  • *.shp – файл с геометрией
  • *.dbf – файл с атрибутивной таблицей набора данных
  • *.shx – индекс пространственных данных
  • *.prj – файл, идентифицирующий систему координат и/или проекцию пространственных данных
  • *.cpg – простой текстовый файл с указанием кодировки содержащихся в файле *.dbf атрибутивных данных

Возможны прочие файлы, которые называются идентично основному файлу *.shp - они тоже важны, нельзя их удалять или перемещать.

Файл проекта QGIS - *.qgs либо файл проекта ArcGIS *.mxd

Файл проекта это текстовый XML файл, в котором хранится вся информация о “проекте”. В частности:

  • данные о наборе слоев и путях к файлам данных;
  • данные о названиях слоев;
  • данные о порядке отображения слоев;
  • стили отображения слоев;
  • группы слоев;
  • система координат и/или проекция проекта
  • данные о листах компоновки карт
  • прочие данные о проекте….

Храните данные аккуратно

Всегда старайтесь соблюдать логику расположения данных. Рекомендуемая структура, позволяющая избежать 99% потенциальных проблем выглядит так:

Обратите внимание, что файл проекта project_name.qgs (QGIS) или project_name.mxd (ArcGIS) желательно располагать на уровень выше, чем любые данные, распределенные по подпапкам проекта. В таком случае будут корректно сохраняться относительные пути к файлам данных в проекте и ваш проект будет легко перенести на новый компьютер всего лишь скопировав (или поместив в архив) корневую папку project_name и все ее содержимое.

Крайне желательно, чтобы путь к папке проекта был как можно короче, содержал как можно меньше символов и не содержал кириллических символов и пробелов.

Настоятельно рекомендуется использовать расположение вида:

C:\gis_projects\lab_01\

Разработчики ПО и авторы курса не несут ответственность за работоспособность путей вида:

C:\Пользователи\Александр Сергеевич\Мои документы\мои гис проекты\первый курс\второй модуль\мой первый проект
