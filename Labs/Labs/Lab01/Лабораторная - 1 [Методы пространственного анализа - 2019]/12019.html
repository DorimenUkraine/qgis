QGIS Browser (в старых версиях QGIS, до версии 3.x)

1.1. QGIS Browser (рис.1) — отдельное приложение для работы с данными в пакете QGIS, являющееся аналогом Проводника в ОС Windows. Основными базовыми функциями QGIS Browser являются:

  • Быстрый просмотр пространственных данных и их свойств;
  • Создание новых пространственных данных (shape-файлов) различной геометрии.

В плане функциональности QGIS Browser уступает своему “визави” из состава ArcGIS — ArcCatalog, однако свою главную функцию — быстрого просмотра геоданных и их свойств, а также создания новых shape-файлов выполняет быстро и эффективно, при этом, например, не умеет удалять классы пространственных данных.

Подготовка данных для картографирования

3.1. Целью данного задания является создание карты населения города Риги по микрорайонам. Вам нужно отобразить на одной карте:

  • общую численность населения каждого из микрорайонов города (каким способом картографического изображения это делается?);
  • плотность населения в каждом из микрорайонов (каким способом картографического изображения это делается?);
  • отдельно выделить официальные границы  административных районов.

Исходя из концепции «географического подхода» после формулирования целей и задач исследования необходимо собрать и подготовить нужные данные.

В Вашей рабочей папке для сегодняшнего задания находятся два shape-файла: riga_neighbourhoods и riga_rayon, а также папка с дополнительными слоями.

Добавьте первый из файлов в Ваш проект любым из известных Вам способов: перетаскиванием файла с расширением .shp из Проводника Windows, с помощью панели Browser, используя Data Source Manager или из меню Layer. Если Вы используете QGIS версии 2.х, воспользуйтесь приложением Browser (см. раздел 1).

Чтобы не тратить время на поиск папки с данными по проекту, подключите данную папку к избранным, нажав правой кнопки мыши на раздел Избранные (Favorites -> Add a Directory...) в встроенной панели Browser внутри программы QGIS и указав путь к необходимой папке с данными, чтобы она отображалась в верхней части каталога (рис.5). В нашем случае это папка с материалами по первой лабораторной MSA_Lab01.

3.4. Плотность населения выражается в количестве человек на единицу площади (как правило на км2). То есть для вычисления нужного значения Вам необходимо знать население каждого района, а также его площадь.

Данные по населению у Вас уже есть, поэтому следующим шагом будет вычисление площади каждого района.

На семинаре по Excel данные значения были предзаданы, но что делать, если геометрические параметры отсутствуют?

3.5. Для того чтобы прописать значения площади в атрибутивную таблицу, необходимо добавить новое пустое поле. По умолчанию в QGIS возможность внесения изменений как в содержимое, так и в сам список атрибутивных полей отключена (это легко проверить, наведя курсор на иконки добавления и удаления атрибутивных полей, которые подсвечены “серым”). Чтобы создать новое поле предварительно необходимо активировать так называемый “режим редактирования” (Editing Mode). Для этого в панели управления атрибутивной таблицей необходимо нажать на
Toggle Editing Mode (рис. 7 ⟹ 1), после чего уже добавить новое поле с помощью соответствующей функции (рис. 7 ⟹ 2).

После этого в новом окне необходимо задать имя (только латинскими буквами!) и тип нового поля (рис. 7 ⟹ 3). Всего предусмотрено шесть типов, на данном этапе обучения Вам необходимо знать три:

  • Whole number — ставится для целочисленных переменных;
  • Decimal number — ставится для переменных с плавающей запятой;
  • Text — ставится для текстовых полей.

После того, как поле создано, сменить его тип нельзя!

Обратите внимание, что в том же всплывающем окне при создании нового поля с типом Decimal number Вам необходимо настроить длину нового поля (Length) и его точность (Precision) для возможности отображения знаков после запятой. Поставьте значение “5” для обоих параметров.

Если Вы точно знаете, что в поле будут храниться только целые числа — ставьте Whole Number. Если сомневаетесь или не знаете — выбирайте тип Decimal Number. Обо всех типах полей можно прочитать в справке либо на форумах QGIS.

Рис. 7. Создание нового поля

Попробуйте создать любое поле (например, с названием “Area_Mercator”) и сразу удалить его с помощью соседней с добавлением поля функции. Зачем нужны все эти манипуляции Вы узнаете чуть позже.

Обратите внимание, что shape-файлы не поддерживают имена полей длиннее 10 символов! Если Вы попытаетесь назвать первое поле «Area_Mercator», QGIS не позволит Вам ввести название длиннее «Area_Merca».

3.6. В настоящий момент системой координат (СК) Вашего фрейма данных является неспроецированная WGS 1984, в чем Вы можете убедиться, зайдя в свойства проекта (Project — Properties — CRS) (рис. 8).

В свойствах проекта поставьте проекцию фрейма WGS 84 / World Mercator ( EPSG:3395 ). Для этого впечатайте название или его часть в строку поиска (рис. 8 ⟹ 1), после чего выберите нужную проекцию (рис. 8 ⟹ 2). Почему Меркаторов так много?

Самый простой способ найти нужную проекцию - запомнить ее уникальный идентификатор - в случае QGIS EPSG-код.

Рис. 8. Установка проекции Меркатора

3.7. Расчет площади в пакете QGIS реализуется с помощью так называемого калькулятора полей (Field Calculator), расположенного на панели управления атрибутивной таблицей справа от иконок добавления и удаления нового поля.

Однако перед собственно расчетом необходимо установить параметры. При выборе проекции WGS 84 / World Mercator ( EPSG:3395 ) на странице настроек геометрических измерений General — Measurements — Ellipsoid (рис. 9 ⟹ 1) автоматически выставляется эллипсоид WGS 84 для точных расчетов площадей и длин - в этом варианте все расчеты уже будут производиться с учетом геодезических, а не спроецированных параметров. Вместо квадратных метров выберите квадратные километры как единицу измерения (рис. 9 ⟹ 2). Нажмите Ок.

В этом аспекте - одно из первых “ключевых” отличий ArcGIS и QGIS, подробнее об этом - на семинаре.

Создание картограммы

4.1. Каждый слой в таблице содержания может иметь только один способ отображения. Поэтому для того, чтобы сделать одновременно и картограмму, и картодиаграмму на основе одного класса данных, необходимо его продублировать. В контекстном меню слоя riga_neighbourhoods выберите «Дублировать» (Duplicate), после чего у Вас появится копия слоя под названием  riga_neighbourhoods_copy.

Если Вы посмотрите на его свойства во вкладке General в параметрах слоя, то название самого shape-файла осталось неизменным. Приписка “copy” появилась только в таблице содержания QGIS.

4.2. Нижний слой с названием  riga_neighbourhoods_copy будет основой для картограммы плотности населения по микрорайонам г. Риги. Для выбора и настройки способа изображения зайдите в свойства слоя (Properties — Symbology) (рис. 13 ⟹ 1). Далее выберите картограмму (Graduated) (рис. 13 ⟹ 2).

Оформление карты

6.1. Финальным этапом любой картографической работы является процесс компоновки карты — расположение всех элементов карты относительно друг друга и относительно картографического изображения.

В рамках данного задания Вам необходимо оформить легенду, добавить название и масштабную линейку. Наиболее важный из  них — легенда, от её корректного оформления зависит правильное прочтение карты. Компоновка и оформление карты в пакете QGIS реализуется с помощью функции Print Layout. Создание нового листа для компоновки и экспорта карты в графическое изображение можно сделать через меню (Project — Print Layout), либо с помощью соответствующей кнопки на базовой панели инструментов (рис. 15 ⟹ 1).

Обратите внимание, что листов может быть более одного (что выгодно отличает QGIS от пакета ArcGIS, где по умолчанию один файл оформления проекта - .mxd - по своей сути соответствует одной картосхеме)

Итак, в проекте может быть неограниченное количество листов с разными наборами слоев, разным размером листа, свойствами отображения и прочими настройками. Поэтому при создании нового листа ему необходимо дать название, предпочтительным является использование латинских букв.

Если вы закрыли окно компоновки, его можно вновь открыть из меню (Project — Layout Manager), либо из менеджера листов компоновки (рис. 17 ⟹ 1). В новом окне будут отображены все созданные ранее листы компоновок (рис. 17 ⟹ 2).

Рис. 17. Менеджер листов компоновок

6.2. Основное окно компоновки листа выглядит следующим образом: слева располагаются элементы, которые могут быть добавлены на лист (включая собственно картографическое изображение)
(рис. 18 ⟹ 1), сверху - настройки печати и экспорта (рис. 16 ⟹ 2) и выравнивания и порядка отображения (рис. 16 ⟹ 3), справа - основное меню настроек каждого отдельного элемента
(рис. 18 ⟹ 4).

Простейшая компоновка листа заключается в добавлении и расположении на листе собственно картографического изображения и следующих элементов оформления: масштабной линейки, “легенды” и названия.

  • изменить единицы измерения (с миль на км);
  • поставить русскоязычные подписи единиц измерения.

Знакомство с ArcCatalog

1.1. ArcCatalog — отдельное приложение для работы с данными в пакете ArcGIS, являющееся аналогом Проводника в ОС Windows (рис. 1). Основными базовыми функциями ArcCatalog являются:

  • Организация системы хранения данных
  • Быстрый просмотр пространственных данных и их свойств
  • Создание новых пространственных данных (в частности shape-файлов) различной геометрии

Все приложения пакета ArcGIS интегрируются между собой, что позволяет, например, для удобства добавлять различные панели инструментов в рабочую среду ArcMap.

Подготовка данных для картографирования

3.1. Целью данного задания является создание карты населения города Риги по микрорайонам. Вам нужно отобразить на одной карте:

  • общую численность населения каждого из микрорайонов города (каким способом картографического изображения это делается?)
  • плотность населения в каждом из микрорайонов (каким способом картографического изображения это делается?)
  • отдельно выделить границы официальных административных районов

Исходя из концепции «географического подхода» после формулирования целей и задач исследования необходимо собрать и подготовить нужные данные.

В Вашей рабочей папке для сегодняшнего задания находятся два shape-файла: riga_neighbourhoods и riga_rayon, а также папка с дополнительными слоями. 

Добавьте первый из файлов в Ваш проект из панели ArcCatalog (см. раздел 2).

3.2. Посмотрите его атрибутивную информацию — какие поля Вы видите?

Перед тем как приступить непосредственно к созданию карты, Вам необходимо прописать все необходимые для этого значения показателей в атрибутивную таблицу нужного слоя.

Для удобства панель атрибутивной таблицы можно встроить в рабочее пространство ArcMap по аналогии с боковой панелью ArcCatalog. Для этого попробуйте передвинуть окно атрибутивной таблицы и из предложенных мест для встраивания выберите наиболее подходящее для Вас.

3.3. Для создания картодиаграммы Вам необходимы данные по населению для каждого из районов города. Эти данные у Вас уже есть в shape-файле riga_neighbourhood (поле Popul).

Имена полей в атрибутивных таблицах ArcGIS могут содержать только латинские символы
(рис. 4 ⟹ 1). При необходимости можно присвоить каждому полю кириллический «псевдоним» (Alias) (рис. 4 ⟹ 2) для упрощения навигации в большом количестве полей. Для вывода окна параметров необходимо нажать правой кнопкой мыши на названии поля и выбрать «Свойства» (Properties).

Рис. 4. Параметры полей атрибутивной таблицы

3.4. Плотность населения выражается в количестве человек на единицу площади (как правило на км2). То есть для вычисления нужного значения Вам необходимо знать население каждого района, а также его площадь.

Данные по населению у Вас уже есть, поэтому следующим шагом будет вычисление площади каждого района.

На семинаре по Excel данные значения были предзаданы, но что делать, если геометрические параметры отсутствуют?

3.5. Для того чтобы прописать значения площади в атрибутивную таблицу, необходимо добавить новое пустое поле. Для этого в меню управления атрибутивной таблицей (рис. 5 ⟹ 1) необходимо нажать на Add Field (рис. 5 ⟹ 2).

После этого в новом окне необходимо задать имя (только латинскими буквами!) и тип нового поля. Всего предусмотрено шесть типов (рис. 5 ⟹ 3), на данном этапе обучения Вам необходимо знать три:

  • Long Integer — ставится для целочисленных переменных;
  • Double — ставится для переменных с плавающей запятой;
  • Text — ставится для текстовых полей.

После того, как поле создано, сменить его тип нельзя!

Если Вы точно знаете, что в поле будут храниться только целые числа — ставьте Long Integer. Если сомневаетесь или не знаете — выбирайте тип Double. Обо всех типах полей можно прочитать в справке ArcGIS.

Рис. 5. Параметры нового поля

Таким образом создайте три новых поля: Area_MC; Area_Lamb; Area_UTM (у всех тип Double).

Обратите внимание, что shape-файлы не поддерживают имена полей длиннее 10 символов! Если Вы попытаетесь назвать первое поле «Area_Mercator», ArcMap автоматически предложит Вам сократить название до «Area_Merca».

3.6. В настоящий момент системой координат (СК) Вашего фрейма данных (View — Data Frame Properties — Coordinate System) является неспроецированная WGS 1984.

Создание картодиаграммы

5.1. Предпоследним этапом в создании карты будет построение картодиаграммы населения микрорайонов города. Вспомните, чем картодиаграмма отличается от картограммы?

Если Вы не переключились в режим компоновки ранее (см. раздел 4.2), сделайте это сейчас.

Это связано с тем, что размеры диаграмм пересчитываются в зависимости от масштаба, поэтому в режиме данных и режиме компоновки итоговое изображение может выглядеть по-разному.

5.2. Для картодиаграмм существует два основных способа установки соответствия между размерами значков и величиной картографируемого показателя.

  • Непрерывная шкала — размер значка диаграммы меняется плавно в соответствии с изменением показателя объекта;
  • Ступенчатая (интервальная) шкала — размер значка диаграммы группируется по заранее заданным интервалам.

Подробнее о шкалах условных знаков говорилось на лекции.

5.3. Для создания картодиаграммы используйте верхний слой riga_neighbourhoods. 

Рис. 10. Настройка картодиаграммы

Методика построения картодиаграммы с использованием интервальной шкалы (Quantities — Graduated Symbols) практически не отличается от методики создания картограммы (см. раздел 4.3).

Необходимо:

  • Выбрать показатель для картографирования;
  • Настроить интервальную шкалу;
  • Установить базовый цвет для диаграмм (либо вручную установить цвет для каждого интервала);
  • Подобрать минимальный и максимальный размер значков (перебором в режиме компоновки) для наилучшей визуализации анализируемых явлений.

Для создания картограммы с использованием непрерывной шкалы в панели способов отображения нужно выбрать Proportional Symbols (рис. 10 ⟹ 1). Так как размер знака диаграммы плавно меняется в зависимости от величины показателя, в данном методе помимо собственно выбора показателя
(рис. 10 ⟹ 2) необходимо настроить только размер и параметры отрисовки минимальной диаграммы (рис. 10 ⟹ 3)! Максимальный размер диаграммы строится автоматически.

Выбор шкалы зависит от нескольких факторов, основными из которых являются особенности конкретного массива данных и задачи, стоящие перед картографом.

На семинарском занятии попробуйте настроить оба метода и выберите тот, который наилучшим образом показывает картографируемое явление. И при этом не забывайте про картограмму на нижележащем слое!

5.4. Следующим шагом необходимо отключить отрисовку районов на картодиаграмме, чтобы стала видна предварительно построенная картограмма. Оформление карты

6.1. Финальным этапом любой картографической работы является процесс компоновки карты — расположение всех элементов карты относительно друг друга и относительно картографического изображения.

В рамках данного задания Вам необходимо оформить легенду, название и масштабную линейку.

6.2. Все инструменты оформления карты находятся в меню «Вставка» (Insert). Наиболее важный из  них — легенда, от её корректного оформления зависит правильное прочтение карты.

ArcMap обладает встроенным конструктором легенды, позволяющим создавать динамические условные обозначения (например, при изменении цвета слоя соответствующий ему условный знак в легенде также поменяет цвет).

После запуска «Мастера легенды» (Legend Wizard) Вам будет предложено выбрать:

  • слои для добавления в легенду (в данном случае добавьте все три слоя) (рис. 11 ⟹ 1);
  • заголовок легенды (и его форматирование);
  • параметры блока легенды (задайте границы (рис. 11 ⟹ 2) и заливку (рис. 11 ⟹ 3), иначе под блоком легенды будет «просвечивать» картографическое изображение);
  • особенности отрисовки отдельных строк легенды;
  • размер интервалов между различными элементами легенды

Последние два пункта можно оставить без изменений.

Запомните типы файлов

Запомните типы файлов, используемых в ГИС и их назначение.

Для наших целей мы будем преимущественно использовать векторные данные в формате ESRI Shapefile. Это один из наиболее универсальных форматов, хотя он достаточно старый и имеет ряд серьезных ограничений. Это все еще один из самых распространенных форматов пространственных данных, по аналогии с файлами *.doc для текстовых документов.

ESRI Shapefile

Это не один файл, а набор файлов, которые по отдельности друг от друга бессмысленны и бесполезны. Эти файлы (все с одинаковым именем но разными расширениями) всегда должны храниться и передаваться вместе, потеря одного из файлов равноценна потере всех файлов.

В составе шейпфайла обычно бывают:

  • *.shp – файл с геометрией
  • *.dbf – файл с атрибутивной таблицей набора данных
  • *.shx – индекс пространственных данных
  • *.prj – файл, идентифицирующий систему координат и/или проекцию пространственных данных
  • *.cpg – простой текстовый файл с указанием кодировки содержащихся в файле *.dbf атрибутивных данных

Возможны прочие файлы, которые называются идентично основному файлу *.shp - они тоже важны, нельзя их удалять или перемещать.

Файл проекта QGIS - *.qgs либо файл проекта ArcGIS *.mxd

Файл проекта это текстовый XML файл, в котором хранится вся информация о “проекте”. В частности:

  • данные о наборе слоев и путях к файлам данных;
  • данные о названиях слоев;
  • данные о порядке отображения слоев;
  • стили отображения слоев;
  • группы слоев;
  • система координат и/или проекция проекта
  • данные о листах компоновки карт
  • прочие данные о проекте….

Храните данные аккуратно

Всегда старайтесь соблюдать логику расположения данных. Рекомендуемая структура, позволяющая избежать 99% потенциальных проблем выглядит так:

Обратите внимание, что файл проекта project_name.qgs (QGIS) или project_name.mxd (ArcGIS) желательно располагать на уровень выше, чем любые данные, распределенные по подпапкам проекта. В таком случае будут корректно сохраняться относительные пути к файлам данных в проекте и ваш проект будет легко перенести на новый компьютер всего лишь скопировав (или поместив в архив) корневую папку project_name и все ее содержимое.

Крайне желательно, чтобы путь к папке проекта был как можно короче, содержал как можно меньше символов и не содержал кириллических символов и пробелов.

Настоятельно рекомендуется использовать расположение вида:

C:\gis_projects\lab_01\

Разработчики ПО и авторы курса не несут ответственность за работоспособность путей вида:

C:\Пользователи\Александр Сергеевич\Мои документы\мои гис проекты\первый курс\второй модуль\мой первый проект
