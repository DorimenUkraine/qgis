Как выполнить задание в...QGIS!

1. Описание данных

1.1. Целью данного задания является анализ плотности дорожной сети (выражающейся в км дорожной сети на кв. км площади территории — км/км2) по следующим зонам:

  • в пределах Садового кольца;
  • от Садового кольца до Третьего Транспортного кольца (ТТК);
  • от ТТК до МКАД.

В результате Вы должны построить картосхему, на которой наглядно будут показаны границы зон, дорожная сеть, а также подписаны (и оформлены) значения плотности дорожной сети для каждой из них.

В папке shapedata Вы найдете два набора геоданных:

  • highway.shp — данные по дорожной сети Москвы
  • bound.shp — границы старой Москвы

1.2. Добавьте все данные в QGIS. Настройте проекцию.

Обратите внимание: для расчета геометрических характеристик объектов выбор правильной проекции имеет решающее значение.

1.3. Попробуйте самостоятельно выполнить данное задание.

Для его выполнения Вам потребуются три новых инструмента геообработки: Symmetrical Difference; Intersect (Vector — Geoprocessing Tools) и Merge Vector Layers (Vector — Data Management Tools).

Если у Вас возникают трудности с его самостоятельным выполнением — воспользуйтесь стандартной инструкцией на последующих страницах. Внимание: далее приведен один из возможных алгоритмов решения задачи, от которого можно - и нужно - отклоняться.

2. Создание полигонов зон

2.1. Cоздайте в папке shapedata папку для Ваших данных (например, my_shp), после чего уже в ней создайте три новых полигональных слоя: zone_sadovoe, tmp_ttk и tmp_mkad (Layer — Create Layer — New Shapefile Layer). Последние два слоя будут временными.

Не забудьте присвоить им систему координат!

2.2. Добавьте новые слои в QGIS (рис. 1 ⟹ 1), с помощью создания объектов в режиме редактирования для каждого слоя (Toggle Editing) (рис. 1 ⟹ 2) нарисуйте в каждом из них соответствующий полигон:

  1. zone_sadovoe — полигон с границей за Садовым кольцом;
  2. tmp_ttk — полигон с границей за ТТК;
  3. tmp_mkad — полигон с границей за МКАД.

Проводите границы полигонов немного ЗА соответствующей трассой (рис. 1 ⟹ 3). Обратите внимание, что в центральной части города полигоны зон должны накладываться друг на друга (рис. 1 ⟹ 4).

Рис. 1. Создание полигонов зон

2.3. После создания всех полигонов сохраните изменения и отключите режим редактирования.

2.4. К настоящему моменту у Вас есть три полигона, накладывающихся друг друга. Чтобы исключить двойной (и тройной) счет при расчетах плотности дорожной сети Вам необходимо вырезать из полигона зоны ТТК накладывающуюся на него зону Садового. Соответственно, из полигона зоны МКАД вырезать две другие (таким образом должны получиться полигоны в форме колец).

Для решения данной задачи используйте инструмент «Симметричная разность» (Vector — Geoprocessing Tools — Symmetrical Difference) — в выходной класс объектов будут записаны неперекрывающиеся области входных и корректирующих пространственных объектов.

2.5. Запустите инструмент.  

Рис. 2. Инструмент “Симметричная разность”

Выберите слой с входными объектами — tmp_ttk (рис. 2 ⟹ 3), в качестве корректирующего слоя выберите zone_sadovoe (рис. 2 ⟹ 4). Создайте выходной слой в папке my_shp и назовите zone_ttk
(рис. 2 ⟹ 3). Не забудьте выбрать в качестве формата выходного класса данных shape-файл!

2.5. Повторите данную операцию для слоя tmp_mkad (корректирующим слоем в этом случае будет tmp_ttk!), назовите полученный класс геоданных zone_mkad.

2.6. Перед началом анализа Вам необходимо свести все три слоя в один. Это можно сделать с помощью инструмента «Слияние векторных слоев» (Vector — Data Management Tools — Merge Vector Layers).

В настройках выберите все три слоя для слияния (zone_mkad, zone_ttk, zone_sadovoe). С некоторой долей вероятности произойдет ошибка. Ее причина — в различающихся атрибутивных таблицах. Иногда - особенно в старых версиях QGIS - условием успешной “склейки” нескольких shape-файлов является полное соответствие их атрибутивных таблиц — не только с точки зрения числа и имен полей, но и их типа и разрядности (!).

Тип и разрядность полей в атрибутивной таблице можно посмотреть в свойствах слоя на вкладке Fields.

Таким образом, в случае возникновения ошибок Вам необходимо привести все атрибутивные таблицы к одному виду. В настоящий момент во всех из них, скорее всего, присутствует лишь поле id. Проверьте его разрядность и в случае несовпадения исправьте ее (удалив и создав новое поле с нужными настройками).  

Один из возможных способов решения проблемы — полное удаление атрибутивных полей во всех трех shape-файлах. Если Вы выберете этот способ — настоятельно рекомендуем после завершения работы инструмента создать поле id и вручную прописать уникальные числовые идентификаторы.

В результате работы инструмента у Вас получится один набор геоданных, включающий в себя три полигона.

3. Подготовка данных для анализа дорожной сети

3.1. Для расчета протяженности дорог внутри каждой из трех зон (и последующего расчета их плотности) необходимо «разделить» объекты дорожной сети между этими зонам. Для этого воспользуйтесь инструментом «Пересечение» (Vector — Geoprocessing Tools — Intersection).

3.2. Запустите инструмент. В разделах «Входной слой» и «Перекрывающий слой» выберите highway и полученный в п. 2.7. слой с полигонами зон. Настройте выходной набор геоданных.

3.3. Откройте атрибутивную таблицу нового слоя. С помощью калькулятора полей создайте новое поле и рассчитайте длины всех объектов с помощью функции $length. В каких единицах измерения посчитаны длины?

Перед выполнением этой операции проверьте проекцию!

4. Расчет плотности дорожной сети по зонам

4.1. Откройте атрибутивную таблицу слоя с полигонами зон. Аналогично п. 3.3 рассчитайте площади каждой из зон.

4.2. Следующая задача — расчет суммарной протяженности сегментов УДС в границах каждой зоны. Эту задачу можно решить несколькими способами.

Один из них — воспользуйтесь следующим алгоритмом: выставите фильтр на каждую зону (контекстное меню слоя —  Filter), с помощью выделения по пространственному расположению (Vector — Research Tools — Select by Location) выделите все линии, лежащие в пределах полигона каждой зоны. Далее с помощью функции вывода статистики (View — Statistical Summary) (рис. 3 ⟹ 1) Вы получите искомые значения протяженности УДС внутри каждой зоны (рис. 3 ⟹ 2) (рис. 3 ⟹ 4). Не забудьте включить опцию выведения статистики только для выделенных объектов, а не всего объектов слоя (рис. 3 ⟹ 3)!

На рис. 3 не обязательно приведен правильный ответ для одной из зон ;)

Рис. 3. Статистика по выделенным объектам

4.3. С помощью полученных значений также рассчитайте плотность дорожной сети для каждой зоны (км/км2).

Запомните типы файлов

Запомните типы файлов, используемых в ГИС и их назначение.

Для наших целей мы будем преимущественно использовать векторные данные в формате ESRI Shapefile. Это один из наиболее универсальных форматов, хотя он достаточно старый и имеет ряд серьезных ограничений. Это все еще один из самых распространенных форматов пространственных данных, по аналогии с файлами *.doc для текстовых документов.

ESRI Shapefile

Это не один файл, а набор файлов, которые по отдельности друг от друга бессмысленны и бесполезны. Эти файлы (все с одинаковым именем но разными расширениями) всегда должны храниться и передаваться вместе, потеря одного из файлов равноценна потере всех файлов.

В составе шейпфайла обычно бывают:

  • *.shp – файл с геометрией
  • *.dbf – файл с атрибутивной таблицей набора данных
  • *.shx – индекс пространственных данных
  • *.prj – файл, идентифицирующий систему координат и/или проекцию пространственных данных
  • *.cpg – простой текстовый файл с указанием кодировки содержащихся в файле *.dbf атрибутивных данных

Возможны прочие файлы, которые называются идентично основному файлу *.shp - они тоже важны, нельзя их удалять или перемещать.

Файл проекта QGIS - *.qgs либо файл проекта ArcGIS *.mxd

Файл проекта это текстовый XML файл, в котором хранится вся информация о “проекте”. В частности:

  • данные о наборе слоев и путях к файлам данных;
  • данные о названиях слоев;
  • данные о порядке отображения слоев;
  • стили отображения слоев;
  • группы слоев;
  • система координат и/или проекция проекта
  • данные о листах компоновки карт
  • прочие данные о проекте….

Храните данные аккуратно

Всегда старайтесь соблюдать логику расположения данных. Рекомендуемая структура, позволяющая избежать 99% потенциальных проблем выглядит так:

Обратите внимание, что файл проекта project_name.qgs (QGIS) или project_name.mxd (ArcGIS) желательно располагать на уровень выше, чем любые данные, распределенные по подпапкам проекта. В таком случае будут корректно сохраняться относительные пути к файлам данных в проекте и ваш проект будет легко перенести на новый компьютер всего лишь скопировав (или поместив в архив) корневую папку project_name и все ее содержимое.

Крайне желательно, чтобы путь к папке проекта был как можно короче, содержал как можно меньше символов и не содержал кириллических символов и пробелов.

Настоятельно рекомендуется использовать расположение вида:

C:\gis_projects\lab_01\

Разработчики ПО и авторы курса не несут ответственность за работоспособность путей вида:

C:\Пользователи\Александр Сергеевич\Мои документы\мои гис проекты\первый курс\второй модуль\мой первый проект
