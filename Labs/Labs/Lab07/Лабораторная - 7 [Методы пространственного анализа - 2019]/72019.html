1. Описание данных

1.1. Целью данного задания является моделирование (иными словами - расчет) поля плотности населения в нескольких районах г. Норильска.

1.2. В материалах Вы найдете shape-файлы с элементами картографической основы для г. Норильска, а также 2 точечных слоя:

  • nrlsk_CENT_build_geocoded.shp – точечный слой с центроидами жилых зданий Центрального района Норильска, полученный в результате геокодирования данных из открытых источников;
  • nrlsk_TALN_build_geocoded.shp – точечный слой с центроидами жилых зданий района Талнах, полученный в результате геокодирования данных из открытых источников.

В чем заключается операция геокодирования? Почему статистика по зданиям представлена в виде точек? Какие методы сбора полевых данных Вы знаете?

1.3. Загрузите набор данных по Центральному району в QGIS (вместе со слоем дорог для возможности навигации). Обратите внимание на проекцию, в которой сохранены точечные данные. Подберите для проекта нужную проекцию UTM.

2. Подготовка данных

2.1. Откройте атрибутивную таблицу слоя nrlsk_CENT_build_geocoded.shp. Обратите внимание на поле Area – в нем прописано значение общей площади для каждого жилого дома на территории Центрального района города. Посчитайте их суммарную площадь.

Почему панель статистики (Statistics Panel) не выводит значение суммы? Какое решение этой проблемы Вы можете предложить? 

Попробуйте решить эту задачу самостоятельно – если не получается – переходите к пункту 2.2.

2.2. Statistics Panel не может вывести сумму для текстовых полей (обратите внимание на тип поля Area в таблице атрибутов – String). Чтобы пересчитать данные в числовой формат необходимо с помощью калькулятора полей создать новое числовое поле (Area_num) (рис. 1 ⟹ 1) и приравнять его к исходному текстовому полю. Однако, это не так просто - простое приравнивание не приведет к нужному результату. За это преобразование отвечает функция to_real. Попробуйте составить требуемое выражение. Почему оно не работает?

Обратите внимание - в атрибутивной таблице в значениях площадей проставлены пробелы в качестве разграничителя разрядов (рис. 1 ⟹ 4) - по этой причине QGIS не может определить числовые значения. Необходимо их удалить. Каким образом это можно сделать?

Рис. 1. Преобразование в числовой формат

Введите в поле поиска функцию replace (рис. 1 ⟹ 2), прочитайте описание и ознакомьтесь с синтаксисом (рис. 1 ⟹ 3). Итоговое выражение, которое Вам необходимо ввести в поле формул, выглядит следующим образом:

to_real( replace( "Area", ' ' , '' ) )   ,

где в двойных кавычках (“ “) фиксируются названия полей, а в одинарных (‘ ‘)  - текстовые фрагменты. Таким образом, пробел фиксируется как ‘ ‘, а значение “пусто” - ‘’.  

После успешного завершения операции рассчитайте и зафиксируйте в блокноте суммарное значение общей площади для всего Центрального района.

2.3. Найдите в интернете информацию о суммарной численности жителей Центрального района Норильска, с ее помощью рассчитайте величину средней обеспеченности общей площадью на каждого жителя района.

2.4. Исходя из полученного значения вычислите расчетную численность населения каждого жилого дома. Для этого с помощью калькулятора полей создайте новое поле Popul (какой тип поля Вы выберете?) и с помощью калькулятора полей разделите общую площадь каждого дома (содержащееся в поле Area_num) на вычисленное в п. 2.3 значение средней обеспеченности. Полученные значения будут играть ключевую роль в построении поля плотности населения.

3. Создание и настройка растровой поверхности

3.1. Растр (по своей сути являющийся картинкой, а еще точнее - таблицей / матрицей) представляет собой непрерывное поле значений, и фактически является результатом перевода дискретной формы распределения явлений в непрерывную. Иными словами – у Вас есть данные по населению всех жилых домов изолированного района города, и Вам необходимо рассчитать непрерывное поле плотности населения, чтобы выяснить, в каких частях города население сконцентрировано, в каких – наоборот, разрежено. Подобная информация может играть ключевую роль, к примеру, при выборе места для размещения определенных категорий объектов обслуживания.

3.2. Большая часть инструментов для работы с растрами сгруппированы во вкладке Raster. В списке  доступных инструментов должен присутствовать Heatmap - другое название - Kernel Density Estimation (или т.н. “тепловая карта”, хотя это выражение считается ругательным в профессиональной среде), по факту являющийся инструментом расчета плотности. Запустите его.

Если в меню Raster отсутствует требуемый инструмент, его можно найти в общей библиотеке набора инструментов Processing. Для этого добавьте известными Вам способами дополнительную панель Processing Toolbox (пример одного из них - на рис. 2, другого - на рис. 4). В окне поиска наберите ключевое слово “density”, после чего во вкладке Interpolation выберите соответствующий инструмент (рис. 2 ⟹ 1).  

Рис. 2. Настройки “тепловой карты”

Запустив инструмент, Вы увидите предупреждение о наличии географической СК в текущем слое
(рис. 2 ⟹ 2). Попробуйте проигнорировать данное предупреждение и посмотреть, что получится :)

В качестве входных данных выберите точечный слой nrlsk_CENT_build_geocoded.shp (рис. 2 ⟹ 2), в который Вы предварительно прописали значение числа жителей каждого дома (поле Popul). В качестве радиуса выставите значение 0,002 град. Оставьте пустым значение выходных данных в нижней части окна, в результате чего будет создан временный класс данных, который хранится в оперативной памяти и будет удален после закрытия программы.

Создание временных классов удобно в случае “экспериментов” по подбору параметров для тех или иных инструментов. Не забывайте сохранять их через настройки экспорта в реальные классы данных.

В разделе Advanced выставите поле, в котором сохранены значения веса (соответственно, поле Popul) (рис. 2 ⟹ 4). Наконец, выставите размер растра в 500 горизонтальных “строк” (“rows”) (рис. 2 ⟹ 3), после чего автоматически выставится число вертикальных “колонок” и размер ячейки. Обратите внимание, что в качестве входных данных для построения растровых поверхностей могут быть использованы только точечные данные.

3.3. После завершения работы инструмента получившийся растр подгрузится автоматически. Исходя из особенностей работы инструмента (о которых Вам рассказывали во вводной части) можно было ожидать округлую форму «зон повышенной плотности» вокруг искомых точечных объектов. С чем связана их овальная форма?

3.4. Причина их необычной формы, как и «странных значений» размера ячеек в настройках инструмента (см. п. 3.2) – в выбранной проекции исходного точечного слоя. При построении растра учитывается не проекция / система координат проекта, а проекция / СК конкретного shape-файла (которая сохранена в файле .prj). Посмотрите текущую проекцию shape-файла nrlsk_CENT_build_geocoded.shp (в параметрах слоя на вкладке Information).

Попробуйте открыть файл .prj в блокноте - только ничего не меняйте.

Для правильного расчета растра точечному набору данных необходимо присвоить не географическую СК, а прямоугольную – в частности UTM нужной зоны.

Пересохраните точечный слой с помощью функции Export - Save Feature As в предварительно созданную папку my_shp.

3.5. Снова запустите инструмент Heatmap. В качестве входных данных выберите созданный в п. 3.4 shape-файл с прописанной проекцией UTM. Обратите внимание на настройки инструмента - в частности размер ячейки теперь задается в метрах. Ключевые параметры при построении растра - радиус “влияния” каждой точки и размер ячейки: попробуйте построить несколько растров с разными значениями (рекомендуемый интервал для радиуса - от 100 до 200 м, для размера ячейки - от 5 до 15 м). Выбор между созданием временных слоев или сохранением всех растров (в папке rasterdata) остается за Вами.

Чтобы лучше понять разницу - настраивайте для каждого из них ступенчатую шкалу (какой способ изображения Вы используете?), для этого во вкладке Simbology в выпадающем списке Render Type выберите Singleband Pseudocolor (рис. 3 ⟹ 1), далее в разделе Interpolation - параметр Discrete 
(рис. 3 ⟹ 2), после этого выберите цветовую шкалу, затем настройте число интервалов по известной Вам схеме (рис. 3 ⟹ 4). Интервалы можно также задавать вручную, внося нужные значения в соответствующие поля (рис. 3 ⟹ 3 - на рисунке приведен случайный пример разбиения, задайте свои границы).

Рис. 3. Настройки визуализации растра

Из всех вариантов оставьте тот, который Вы считаете наиболее адекватным пространственному распределению явления.

Все ухищрения с настройками радиуса и размера ячеек направлены на то, чтобы, во-первых, получить уровень генерализации, адекватный распределению картографируемого явления (уменьшить число “шумов”), во-вторых, максимально широкий пространственный охват для избежания “дырок” в поверхности. Как Вы заметили, пространственный охват фактически увеличивается с увеличением радиуса.

Если вы экспериментировали с временными слоями, не забудьте их сохранить!

4. Обрезка растровой поверхности

4.1. Добавьте слой raster_CENT_extent - это полигон, обозначающий границы растровой поверхности. С большой долей вероятности Ваш растр выступает за его границы. Чтобы этого не происходило, необходимо “обрезать” растр.

4.2. Перед этим проверьте СК данного shape-файла, при необходимости перепроецируйте его.

4.3. Для выполнения следующей операции Вам также понадобится панель Processing Toolbox. Второй способ ее добавить (первый - на рис. 2) - через соответствующий пункт в верхнем меню: Processing - Processing Toolbox (рис. 4 ⟹ 1). В строке поиска введите “clip” (рис. 4 ⟹ 2), в разделе GDAL/OGR выберите инструмент Clip Raster by Mask Layer (рис. 4 ⟹ 3). Запустите его.

Рис. 4. Панель инструментов геообработки

4.4. В новом окне настройте входные слои (рис. 5 ⟹ 1), далее отметьте пункт “Сохранить разрешение выходного растра” (опционально) (рис. 5 ⟹ 2), наконец, настройте выходную растровую поверхность (как и все остальные - сохраните ее в папку rasterdata) (рис. 5 ⟹ 3).

Рис. 5. Инструмент “обрезки” растра

(5. Сглаживание растровой поверхности)

5.1. Настройте параметры визуализации растра аналогично пункту 3.5 (как вариант - через копирование стилей).

5.2. Если у Вас получилась излишне подробная поверхность, изобилующая небольшими «шумами», а “махинации” с размером ячейки и радиусом не помогают, можно ее дополнительно сгладить (генерализовать) для акцентирования внимания на основных пространственных закономерностях в данном масштабе.

5.3. Для генерализации растровых данных предусмотрено большое число различных инструментов, входящих в состав расширенных библиотек. Один из самых простых – инструменты Simple Filter или Gaussian Filter (с последним из них многие могут быть знакомы по обработке фотографий - часто он называется “размытие по Гауссу”). Чтобы его вызвать (а также посмотреть весь список доступных инструментов сглаживания растров) наберите в поисковой строке панели геообработки “filter”. В библиотеке SAGA представлено большое число подобных инструментов.

Вспомните, что когда Вы устанавливали QGIS - была опция поставить QGIS с GRASS GIS - теперь Вы видите инструменты из этой библиотеки в числе прочих.

Рис. 6. Инструменты сглаживания растра

Запустите инструмент Simple Filter или Gaussian Filter. В качестве входных данных используется полученная в п. 4.4 растровая поверхность, настройте выходной набор растровых данных.  

Данный инструмент работает по принципу т.н. «скользящего окна» (значение Square параметра Search Mode), усредняя значения между соседними ячейками растра. Эффективность его работы напрямую зависит от выбранного ранее размера ячейки растра (Cell Size), если он слишком маленький, изменений может быть не заметно. В этом случае увеличивайте радиус сглаживания (Search Radius) в настройках инструментов. И не забывайте про возможность вызова справки по каждому инструменту - без нее не всегда понятно назначение отдельных опций.

6. Построение изолиний

6.1. После всех манипуляций с растрами настройте финальное оформление итогового варианта (при этом оптимальным является использование равноинтервальной шкалы).

Чтобы убрать “дырки” в растре, а также его “рваные” края, настройте цвет полигона raster_CENT_extent в соответствии с цветом первого интервала растровой поверхности. Дополнительно можете добавить границу растровой поверхности, продублировав данный полигональный слой и отключив заливку.

6.2. Один из способов повышения читаемости границ между соседними градациями цветов на растровом изображении – построение изолиний, представленных отдельным линейным shape-файлом. Инструмент, отвечающий за их создание – Contour (Raster - Extraction - Contour). 

Рис. 7. Инструмент создания изолиний

6.3. Запустите инструмент. Настройте входные (итоговый растр) и выходные (линейный shape-файл, сохраните его в папку my_shp) данные (рис. 7 ⟹ 3). В поле Interval Between Contour Lines введите значение интервала с предварительно настроенного в п. 6.1 итогового растрового изображения. В случае, если Вам необходимо отдельно ввести начальное значение (к примеру, “100” для значений изолиний с шагом “200”, таким образом, “300”, “500”, “700” и т.д.), задайте его в соответствующем разделе (рис. 7 ⟹ 2).

6.3. Настройте оформление нового набора данных, выбрав цвет и толщину линии. При необходимости Вы можете удалить лишние линии в режиме редактирования.

7. Оформление итоговой картосхемы

7.1. По аналогии постройте поле плотности населения для района Талнах, добавив в Ваш проект соответствующие наборы данных – nrlsk_TALN_build_geocoded.shp, raster_TALN_extent.shp. При оформлении этого растра обратите внимание, что интервалы в классификации должны совпадать с ранее настроенными интервалами для Центрального района. При необходимости подберите необходимые значения для обоих поверхностей, в этом случае не забудьте перестроить изолинии.

Изучаемые районы города находятся на значительном удалении друг от друга. Для быстрой навигации можно использовать функцию Zoom to Layer в контекстном меню слоя (активируемом правой кнопкой мыши).

7.2. Перейдите в режим компоновки. В рамках настоящего задания предлагается следующая компоновка для итоговой картосхемы – лист в произвольной ориентации делится на 2 части – в одной представлен фрейм с визуализацией данных по Центральному району, в другой – по району Талнах.

Загрузите в проект слои картографической основы из папки shapedata. Подумайте, как можно на одном картографическом изображении совместить и векторные слои основы, и растровую поверхность?

7.3. Обратите особое внимание на оформление легенды (в т.ч. на используемые Вами единицы измерения плотности населения). Как выполнить задание в...ArcGIS!

1. Описание данных

1.1. Целями данного задания является моделирование (иными словами - расчет) поля плотности населения в нескольких районах г. Норильска. Перед началом работы отключите Background Processing в меню Geoprocessing – Geoprocessing Options.

1.2. В материалах Вы найдете shape-файлы с элементами картографической основы для г. Норильска, а также 2 точечных слоя:

  • nrlsk_CENT_build_geocoded.shp – точечный слой с центроидами жилых зданий Центрального района Норильска, полученный в результате геокодирования данных из открытых источников;
  • nrlsk_TALN_build_geocoded.shp – точечный слой с центроидами жилых зданий района Талнах, полученный в результате геокодирования данных из открытых источников.

В чем заключается операция геокодирования? Почему статистика по зданиям представлена в виде точек? Какие методы сбора полевых данных Вы знаете?

1.3. Загрузите набор данных по Центральному району в ArcMap (вместе со слоем дорог для возможности навигации). Обратите внимание на проекцию, в которой сохранены точечные данные. Подберите для фрейма нужную проекцию UTM.

2. Подготовка данных

2.1. Откройте атрибутивную таблицу слоя nrlsk_CENT_build_geocoded.shp. Обратите внимание на поле Area – в нем прописано значение общей площади для каждого жилого дома на территории Центрального района города. Посчитайте их суммарную площадь.

Почему функция Statistics для этого поля неактивна? Какое решение этой проблемы Вы можете предложить?

Попробуйте решить эту задачу самостоятельно – если не получается – переходите к пункту 2.2.

2.2. Функция Statistics неактивна для текстовых полей (обратите внимание на тип поля Area в таблице атрибутов – String). Чтобы пересчитать данные в числовой формат необходимо создать новое числовое поле и с помощью калькулятора полей приравнять его к исходному текстовому полю. Таким образом, создайте поле Area_num и пропишите в нем общую площадь каждого жилого здания. Посчитайте и зафиксируйте в блокноте суммарное значение показателя для всего Центрального района.

Если калькулятор поля выдает ошибку при попытке приравнять Area_num к Area, значит, на Вашем компьютере в качестве разделителя целой и дробной части стоит запятая. При вычислении поставьте следующее выражение:

replace( [Area] , “.” , “,” )

2.3. Найдите в интернете информацию о суммарной численности жителей Центрального района Норильска, с ее помощью рассчитайте величину средней обеспеченности общей площадью на каждого жителя района.

2.4. Исходя из полученного значения вычислите расчетную численность населения каждого жилого дома. Для этого создайте новое поле Popul (какой тип поля Вы выберете?) и с помощью калькулятора полей разделите общую площадь каждого дома (содержащееся в поле Area_num) на вычисленное в
п. 2.3 значение средней обеспеченности. Полученные значения будут играть ключевую роль в построении поля плотности населения.

3. Создание растровой поверхности

3.1. Растр (по своей сути являющийся картинкой) представляет собой непрерывное поле значений, и фактически является результатом перевода дискретной формы распределения явлений в непрерывную. Иными словами – у Вас есть данные по населению всех жилых домов изолированного района города, и Вам необходимо рассчитать непрерывное поле плотности населения, чтобы выяснить, в каких частях города население сконцентрировано, в каких – наоборот, разрежено. Подобная информация может играть ключевую роль, к примеру, при выборе места для размещения определенных категорий объектов обслуживания.

3.2. Большая часть инструментов для работы с растрами сгруппированы в модуле Spatial Analyst Tools, который не входит в базовый набор модулей ArcGIS (при этом может быть активирован в триал-версии) (рис. 1 ⟹ 1). В состав данного модуля входит группа инструментов расчета плотности, наиболее простым является Point Density. Запустите его из списка ArcToolbox (рис. 1 ⟹ 2).

Рис. 1. Инструмент Point Density

В качестве входных данных выберите точечный слой nrlsk_CENT_build_geocoded.shp (рис. 1 ⟹ 3), в который Вы предварительно прописали значение числа жителей каждого дома (поле Popul). Настройте выходные данные (рис. 1 ⟹ 4), указав в качестве места их сохранения директорию my_shp (которую необходимо создать самостоятельно) в папке к этому заданию и присвоив название по своему усмотрению. Остальные поля оставьте со значениями по умолчанию. Обратите внимание на необычное значение параметра Cell Size (размер ячейки растра). С чем оно может быть связано? Нажмите Ок.

Обратите внимание, что в качестве входных данных для построения растровых поверхностей могут быть использованы только точечные данные.

Подробнее о работе инструмента см. http://pro.arcgis.com/ru/pro-app/tool-reference/spatial-analyst/point-density.htm

3.3. После завершения работы инструмента получившийся растр подгрузится автоматически. Исходя из особенностей работы инструмента (о которых Вам рассказывали во вводной части) можно было ожидать округлую форму «зон повышенной плотности» вокруг искомых точечных объектов. С чем связана их овальная форма?

3.4. Причина их необычной формы, как и «странных значений» размера ячеек в настройках инструмента (см. п. 3.2) – в выбранной проекции исходного точечного слоя. При построении растра учитывается не проекция / система координат фрейма данных, а проекция / СК конкретного shape-файла (которая сохранена в файле .prj). Посмотрите текущую проекцию shape-файла nrlsk_CENT_build_geocoded.shp (в параметрах слоя на вкладке Source). Для правильного расчета растра точечному набору данных необходимо присвоить не географическую СК, а проективную – в частности UTM нужной зоны.

Попробуйте открыть файл .prj в блокноте - только ничего не меняйте.

Данная операция осуществляется с помощью инструмента Project (в наборе Data Management Tools – Projections and Transformations) (рис. 2 ⟹ 1). Входные данные – точечный слой nrlsk_CENT_build_geocoded.shp (рис. 2 ⟹ 2). Определите нужную проекцию выходных данных (подходящую зону UTM) (рис. 2 ⟹ 3). Не забудьте выбрать место хранения и имя выходного shape-файла (внутри папки my_shp)! Запустите инструмент.

Рис. 2. Инструмент Project

Слой с новыми данными добавится автоматически после завершения работы инструмента. Проверьте его проекцию / СК на вкладке Source.

3.5. Второй нюанс при построении растра (см. растр, получившийся в п. 3.2) – его пространственный охват, который автоматически рассчитывается исходя из крайних точек входного набора данных. В результате этого растр «обрезается» слишком близко к исходным данным. Для целей визуализации часто необходимо его расширить на некоторое расстояние за их пределы. Для настройки подобных частных параметров в большинстве инструментов предусмотрены т.н. «настройки среды» (Environmental Settings).

Снова запустите инструмент Point Density (рис. 3 ⟹ 1). В качестве входных данных выберите созданный в п. 3.4 shape-файл с прописанной проекцией UTM (рис. 3 ⟹ 2). Благодаря этому полигональному слою границы рассчитываемого растра будут расширены и он не будет казаться «обрезанным». Обратите внимание на настройки инструмента (рис. 3 ⟹ 5), в частности размер ячейки теперь задается в метрах. Поставьте значение 40 метров, чтобы оптимизировать вес выходного растра. Ознакомьтесь с оставшимися настройками в описании инструмента в справочной системе ESRI (ссылка приведена в п. 3.2).

Ответьте на вопрос: в чем связь между весом растра и его размером ячейки?

3.6. Наиболее важный, и одновременно субъективный параметр при настройке инструмента – т.н. «Радиус учитываемой области» (рис. 3 ⟹ 5). От него будет полностью зависеть получающаяся картинка, значение параметра подбирается исходя из требуемого уровня генерализации, масштаба и представлений картографа о территории и изучаемом явлении. Попробуйте создать несколько растров с разными значениями радиуса (в диапазоне от 150 до 250 м). Сравните их, и выберите оптимальный на Ваш взгляд. Остальные удалите из проекта.

3.7. Все получающиеся с помощью инструментов геообработки растры имеют прямоугольную форму. Зачастую требуется «обрезать» растр по заранее определенному полигону – в нашем случае по слою raster_CENT_extent.shp – как в целях дальнейшей работы, так и итоговой картографической визуализации. Для «обрезки» растра по другому полигональному слою в ArcGIS предусмотрен инструмент Clip (Data Management Tools – Raster – Raster Processing).

Обратите внимание, что данный инструмент отличается от уже знакомого Вам инструмента Clip, предназначенного для обрезки векторных данных!

Запустите инструмент (рис. 4 ⟹ 1). В качестве входных данных выберите Ваш растр (рис. 4 ⟹ 2), в качестве слоя, по которому он будет обрезаться – shape-файл raster_CENT_extent.shp (рис. 4 ⟹ 3), обязательно поставьте «галочку» напротив пункта Clipping Geometry!

3.8. В результате всех манипуляций Вы получили растровую поверхность, которая наглядно отражает пространственное распределение значений плотности населения по территории г. Норильска.

(4. Сглаживание растровой поверхности)

4.1.Настройте параметры визуализации растра – в параметрах слоя на вкладке Display в разделе Resample during display выставите Bilinear Interpolation. Обратите внимание, что изменилось.

4.2. Если у Вас получилась излишне подробная поверхность, изобилующая небольшими «шумами», ее необходимо сгладить (генерализовать) для акцентирования внимания на основных пространственных закономерностях в данном масштабе.

4.3. Для генерализации растровых данных предусмотрено большое число различных инструментов, один из самых простых, при этом наиболее эффективных – инструмент Filter (Spatial Analyst Tools – Neighborhood). Запустите инструмент. В качестве входных данных используется полученная в п. 3.8 растровая поверхность, настройте выходной набор растровых данных, в поле «Тип фильтра» выберите Low Pass. При необходимости можно запускать инструмент несколько раз для достижения оптимального результата.

Данный инструмент работает по принципу т.н. «скользящего окна», усредняя значения между соседними ячейками растра. Эффективность его работы напрямую зависит от выбранного ранее размера ячейки растра (Cell Size).

Запускать инструменты можно с помощью панели поиска, активируемой на главной панели инструментов (справа от окна масштаба).

Научитесь пользоваться архиватором

Запомните, для того чтобы работать с файлами из архива, его содержимое сначала надо распаковать в какую-либо папку на диске. В противном случае, абсолютное большинство программ, включая QGIS, не сможет работать с файлами напрямую из архива.

В Windows обычно нужно нажать на архиве правой кнопкой и выбрать распаковать.

В macOS чаще всего достаточно два раза кликнуть на архив для распаковки.

Рекомендуемые архиваторы для Windows:

  • 7-zip (бесплатный, архивация и разархивация): http://www.7-zip.org/
  • WinRAR (коммерческий, архивация и разархивация): http://www.rarlab.com/download.htm

Рекомендуемые архиваторы для macOS:

  • Keka (бесплатный, архивация и разархивация): http://www.kekaosx.com/en/
  • The Unarchiver (бесплатный, только разархивация, автоматическое определение кодировки имен файлов в zip архивах): https://itunes.apple.com/ru/app/the-unarchiver/id425424353

Запомните типы файлов

Запомните типы файлов, используемых в ГИС и их назначение.

Для наших целей мы будем преимущественно использовать векторные данные в формате ESRI Shapefile. Это один из наиболее универсальных форматов, хотя он достаточно старый и имеет ряд серьезных ограничений. Это все еще один из самых распространенных форматов пространственных данных, по аналогии с файлами *.doc для текстовых документов.

ESRI Shapefile

Это не один файл, а набор файлов, которые по отдельности друг от друга бессмысленны и бесполезны. Эти файлы (все с одинаковым именем но разными расширениями) всегда должны храниться и передаваться вместе, потеря одного из файлов равноценна потере всех файлов.

В составе шейпфайла обычно бывают:

  • *.shp – файл с геометрией
  • *.dbf – файл с атрибутивной таблицей набора данных
  • *.shx – индекс пространственных данных
  • *.prj – файл, идентифицирующий систему координат и/или проекцию пространственных данных
  • *.cpg – простой текстовый файл с указанием кодировки содержащихся в файле *.dbf атрибутивных данных

Возможны прочие файлы, которые называются идентично основному файлу *.shp - они тоже важны, нельзя их удалять или перемещать.

Файл проекта QGIS - *.qgs либо файл проекта ArcGIS *.mxd

Файл проекта это текстовый XML файл, в котором хранится вся информация о “проекте”. В частности:

  • данные о наборе слоев и путях к файлам данных;
  • данные о названиях слоев;
  • данные о порядке отображения слоев;
  • стили отображения слоев;
  • группы слоев;
  • система координат и/или проекция проекта
  • данные о листах компоновки карт
  • прочие данные о проекте….

Обратите внимание, что файл проекта project_name.qgs (QGIS) или project_name.mxd (ArcGIS) желательно располагать на уровень выше, чем любые данные, распределенные по подпапкам проекта. В таком случае будут корректно сохраняться относительные пути к файлам данных в проекте и ваш проект будет легко перенести на новый компьютер всего лишь скопировав (или поместив в архив) корневую папку project_name и все ее содержимое.

Крайне желательно, чтобы путь к папке проекта был как можно короче, содержал как можно меньше символов и не содержал кириллических символов и пробелов.

Настоятельно рекомендуется использовать расположение вида:

C:\gis_projects\lab_01\

Разработчики ПО и авторы курса не несут ответственность за работоспособность путей вида:

C:\Пользователи\Александр Сергеевич\Мои документы\мои гис проекты\первый курс\второй модуль\мой первый проект
