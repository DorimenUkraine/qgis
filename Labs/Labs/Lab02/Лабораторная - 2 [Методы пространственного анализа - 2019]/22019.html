1. Удаление полей атрибутивной таблицы

1.1. На прошлом занятии Вы познакомились с атрибутивной таблицей слоя, научились добавлять новые поля, выбирать типы данных (вспомните, какие типы данных есть в QGIS?), пользоваться калькулятором полей, в частности считать основные геометрические характеристики объектов (в частности - площадь).

Как правило в открытых геоданных присутствует мало атрибутивной информации, поэтому наиболее распространенной задачей является импорт табличных данных в ГИС. Возможности QGIS позволяют присоединять к существующим пространственным объектам различные типы статистической информации на основе сопоставления так называемых ключевых полей.

1.2. В данных для этого задания Вы найдёте несколько shape-файлов. Добавьте USA_states.shp любым известным Вам способом (рис. 1 ⟹ 1). Этот класс данных был получен с ресурса GADM, предоставляющего данные по административно-территориальному делению стран мира. Ознакомьтесь с его атрибутивной таблицей.

Рис. 1. Удаление полей в QGIS

Хорошо видно, что большая часть полей не представляют никакого интереса: они либо пусты, либо содержат служебную информацию (вроде поля с указанием страны - очевидно, что в случае со штатами США значение поля будет единым для всех объектов). Перед началом работы удалите их, чтобы не перегружать атрибутивную таблицу ненужными данными.

Для удаления поля атрибутивной таблицы необходимо активировать режим редактирования в верхнем меню атрибутивной таблицы (рис. 1 ⟹ 2) и в правой части этого же меню выберите пункт «Удалить Поле» (Delete Field) (рис. 1 ⟹ 3). В новом окне выделите названия всех полей, которые необходимо удалить (рис. 1 ⟹ 4).

Удалите таким образом все поля кроме названий штатов США (поле NAME_1). Отключите режим редактирования, сохранив изменения.

После сохранения изменений возможности восстановить поля не будет!

2. Соединение таблиц

2.1. Помимо shape-файлов в архиве с данными для сегодняшнего занятия находится таблица US_stats.xlsx. Откройте её в MS Excel. В ней представлены данные по совокупному доходу граждан и численности населения по штатам США за 2006 и 2012 гг.

Эта информация была получена с интернет-портала Бюро Экономического Анализа (Bureau of Economic Analysis, U.S. Department of Commerce) и минимально преобразована средствами Excel.

Имея пространственные данные по штатам США (в формате shape-файлов) и статистические данные (в формате MS Excel) по ним же, можно получить готовый shape-файл со всей доступной статистической информацией.

2.2. Для объединения двух таблиц (атрибутивной и таблицы Excel) необходимо однозначное основание для сопоставления, другими словами - т.н. “ключевое поле” в обеих таблицах.

Рис. 2. Соединение таблиц в QGIS

2.3. Предварительно необходимо загрузить таблицу Excel в QGIS. Для этого просто “перетяните” файл из проводника в таблицу содержания, либо в контекстном меню таблицы в браузере нажмите “Добавить слой” (Add Layer) (рис. 2 ⟹ 1).

Обратите внимание, сколько листов в таблице Excel? Попробуйте продублировать лист Data и заново добавить таблицу в QGIS. 

Инструмент “Соединение” (Joins) находится в окне параметров слоя (рис. 2 ⟹ 2). Нажмите кнопку добавления новой “связи” (в форме “плюса”) (рис. 2 ⟹ 3). В новом окне Вам предлагается выбрать, во-первых, присоединяемый слой (в Вашем случае - предварительно подгруженная таблица Excel), во-вторых, поле для сопоставления в присоединяемом слое, и в-третьих, поле существующей атрибутивной таблицы, на основании которого будут добавляться данные из таблицы Excel
(рис. 2 ⟹ 4). В Вашем случае это поле NAME_1.

Обратите внимание, что необходимо удалить всю информацию в строке “Пользовательский префикс поля” (Custom Field Name Prefix) (рис. 2 ⟹ 5), иначе ко всем полям присоединяемой таблицы будет добавлено название файла Excel.

Рис. 3. Параметры полей в QGIS

2.4. Теперь к полям атрибутивной таблицы слоя USA_states добавились все столбцы из таблицы Excel. При этом обратите внимание, что в атрибутивной таблице отсутствует поле States, осталось лишь поле NAME_1 и поля со статистическими данными (рис. 3 ⟹ 2).

После соединения таблиц QGIS не добавляет новые поля в существующую атрибутивную таблицу слоя! Он создаёт связь между ней и файлом MS Excel, что хорошо видно в свойствах слоя на вкладке Fields (рис. 3 ⟹ 1): у вновь присоединенных полей в первом столбце присутствует символ, обозначающий факт соединения (рис. 3 ⟹ 2). В теории это может негативно сказываться на быстродействии и стабильности будущих операций, поэтому после соединения таблиц рекомендуется создать новый shape-файл с единой атрибутивной таблицей.

2.5. Для создания нового shape-файла на основе существующего используется инструмент «Экспорт» (Export - Save Feature As), который находится в контекстном меню слоя (рис. 4 ⟹ 1).

Рис. 4. Экспорт shape-файла

После запуска данного инструмента Вам будет предложено выбрать систему координат (СК) (так как проекция фрейма не менялась, выберите текущую СК) (рис. 4 ⟹ 3), место сохранения, название (например, USA_states_UPD) и тип нового файла геоданных (рис. 4 ⟹ 2) - в последнем случае Вам требуется формат ESRI shapefile. Обратите внимание - присутствуют ли в списке форматов знакомые Вам?

В названиях полей и shape-файлов вместо пробелов всегда используйте знак нижнего подчеркивания «_»! Так QGIS будет корректнее работать с данными.

Обратите внимание также на кодировку слоя - однако эта настройка имеет значение только в случае, когда в атрибутивной таблице присутствует кириллица. В Вашем случае ее нет - поэтому смело выбирайте базовую кодировку UTF-8 (рис. 4 ⟹ 4).  Если в этом же всплывающем окне у Вас был отмечен пункт “Добавить слой на карту” (Add saved file to map), то после сохранения новый слой будет автоматически добавлен в проект.

Также обратите внимание на раздел Select Fields to export - можно выбрать те поля, которые необходимо экспортировать.

4. Построение картограмм

4.1. Создание двух однотипных карт не подразумевает двойную работу по построению картограмм и компоновке. Вам необходимо подготовить один полный набор слоев, включающий в себя готовые данные для обеих карт.

Для этого создайте дубликат слоя USA_states_UPD.

В контекстном меню слоя нажмите «Дублировать» (Duplicate Layer). При этом создается просто дополнительный слой визуализации того же самого shape-файла! Их отображение может быть настроено различным образом, однако, в случае внесения и сохранения правок в режиме редактирования (к примеру, изменения атрибутивной таблицы) все изменения незамедлительно отобразятся и на втором слое! Попробуйте поэкспериментировать, предварительно создав копию всех данных. Обратите внимание, что операция дублирования слоя не создает новую копию файла и исходными данными на жестком диске!

Чтобы в дальнейшем облегчить работу с таблицей содержания припишите к названиям каждого из слоев год, для которого будет строиться картограмма («2012» и «2006» соответственно) (рис. 6 ⟹ 1).

Рис. 6. Переименование слоев

Для этого выделите название слоя и нажмите F2, либо переименуйте его в свойствах слоя (Properties — Source), либо воспользовавшись функцией Rename Layer в контекстном меню слоя (рис. 6 ⟹ 2).

Последовательность слоёв в таблице содержания не имеет значения.

Так как Вам нужно разбить весь массив данных по одним карманам с одинаковыми параметрами отрисовки, необходимо будет настроить разбивку в ручном режиме. Но прежде чем приступить к определению интервалов, нужно сопоставить оба набора данных (подушевой доход и за 2006, и за
2012 гг.) и выявить минимальное и максимальное значения показателя для обоих массивов.

Получить статистику по полям можно с помощью инструмента «Статистика» (View — Statistical Summary) (рис. 7 ⟹ 1). Запустив его, Вы можете с помощью выпадающего списка отобразить статистику для любого поля в атрибутивной таблице любого слоя (рис. 7 ⟹ 2). Вас интересуют значения минимума и максимума для двух полей со значением подушевого дохода (рис. 7 ⟹ 3). Выпишите их.

Запомните типы файлов

Запомните типы файлов, используемых в ГИС и их назначение.

Для наших целей мы будем преимущественно использовать векторные данные в формате ESRI Shapefile. Это один из наиболее универсальных форматов, хотя он достаточно старый и имеет ряд серьезных ограничений. Это все еще один из самых распространенных форматов пространственных данных, по аналогии с файлами *.doc для текстовых документов.

ESRI Shapefile

Это не один файл, а набор файлов, которые по отдельности друг от друга бессмысленны и бесполезны. Эти файлы (все с одинаковым именем но разными расширениями) всегда должны храниться и передаваться вместе, потеря одного из файлов равноценна потере всех файлов.

В составе шейпфайла обычно бывают:

  • *.shp – файл с геометрией
  • *.dbf – файл с атрибутивной таблицей набора данных
  • *.shx – индекс пространственных данных
  • *.prj – файл, идентифицирующий систему координат и/или проекцию пространственных данных
  • *.cpg – простой текстовый файл с указанием кодировки содержащихся в файле *.dbf атрибутивных данных

Возможны прочие файлы, которые называются идентично основному файлу *.shp - они тоже важны, нельзя их удалять или перемещать.

Файл проекта QGIS - *.qgs либо файл проекта ArcGIS *.mxd

Файл проекта это текстовый XML файл, в котором хранится вся информация о “проекте”. В частности:

  • данные о наборе слоев и путях к файлам данных;
  • данные о названиях слоев;
  • данные о порядке отображения слоев;
  • стили отображения слоев;
  • группы слоев;
  • система координат и/или проекция проекта
  • данные о листах компоновки карт
  • прочие данные о проекте….
